## ⏰ Cron 配置

**配置命令：**
```bash
*/30 * * * * cd /home/windows/.openclaw/agents/personal-assistant/workspace/记忆库/memory-system/plugins && timeout 300 python3 scheduled_save.py >> logs/scheduled_save.log 2>&1
```

**说明：**
- `*/30` - 每 30 分钟执行一次
- `timeout 300` - 超时限制 5 分钟
- `>> logs/scheduled_save.log` - 日志保存位置

---

## 🎯 工作原理

```
每 30 分钟
   ↓
读取 OpenClaw 会话文件（.jsonl）
   ↓
提取最近 50 条对话
   ↓
保存到向量库（ChromaDB）
   ↓
记录日志
```

## ✅ 功能特点

### 优势

| 特性 | 说明 |
|------|------|
| **不需要 User Token** | ✅ 直接读取本地会话文件 |
| **开机自启动** | ✅ Cron 是系统服务，开机自动运行 |
| **独立运行** | ✅ 不依赖 OpenClaw 或 AI 助手 |
| **永久有效** | ✅ 一次配置，永久生效 |
| **后台运行** | ✅ 不影响回复速度 |
| **可靠稳定** | ✅ Linux 系统级服务 |

### 对比方案

| 方案 | Token 需求 | 开机自启 | 独立性 | 可靠性 |
|------|-----------|---------|--------|--------|
| **Cron 方案** | ✅ 不需要 | ✅ 是 | ✅ 完全独立 | ✅ 系统级 |
| Heartbeat 方案 | ❌ 需要 | ❌ 否 | ❌ 依赖框架 | ⚠️ 中等 |
| 实时保存方案 | ❌ 需要 | ❌ 否 | ❌ 依赖代码 | ⚠️ 中等 |

## 📞 故障排查

### 问题 1：没有保存对话

**检查：**
```bash
# 1. 查看日志
tail -f logs/scheduled_save.log

# 2. 检查 Cron 配置
crontab -l

# 3. 检查 Cron 服务
systemctl status cron
```

### 问题 2：保存速度慢

**原因：** 加载 embedding 模型需要时间

**解决：** 增加 timeout 时间
```bash
# 编辑 crontab
crontab -e

# 修改 timeout 值（当前 300 秒）
timeout 600 python3 scheduled_save.py
```
