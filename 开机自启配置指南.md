# 记忆库系统 - 开机自启配置指南

## ✅ 已完成的配置

### 1. Cron 服务自启

**配置位置：** `~/.bashrc`

**配置内容：**
```bash
# 自动启动 Cron 服务（记忆库定时任务依赖）
if [ -f /etc/init.d/cron ]; then
    sudo service cron start 2>/dev/null || service cron start 2>/dev/null || true
fi
```

**说明：**
- 每次打开新的 terminal 时自动启动 cron 服务
- 确保定时任务能够正常执行

---

### 2. HuggingFace 镜像源

**配置位置：** `~/.bashrc`

**配置内容：**
```bash
# HuggingFace 镜像源（用于下载嵌入模型）
export HF_ENDPOINT="https://hf-mirror.com"
```

**说明：**
- 解决国内访问 HuggingFace 网络问题
- 确保嵌入模型能够正常加载

---

### 3. 环境变量

**配置位置：** `~/.bashrc`

**配置内容：**
```bash
# 记忆库系统目录
export MEMORY_SYSTEM_DIR="$HOME/.openclaw/agents/personal-assistant/workspace/记忆库/memory-system"
```

**说明：**
- 方便在其他脚本中使用记忆库系统目录

---

### 4. Crontab 定时任务

**配置内容：**
```bash
# 每分钟执行一次归档脚本
* * * * * cd $MEMORY_SYSTEM_DIR && /usr/bin/python3 src/archiver_runner.py >> $MEMORY_SYSTEM_DIR/logs/archiver.log 2>&1

# 每天凌晨 00:00 执行每日备份
0 0 * * * cd $MEMORY_SYSTEM_DIR && /usr/bin/python3 src/daily_backup_runner.py >> $MEMORY_SYSTEM_DIR/logs/daily_backup.log 2>&1
```

**说明：**
- 每分钟自动获取并存储新消息
- 每天凌晨自动备份并推送到 GitHub

## 🔧 故障排查

### 问题 1：Cron 服务未启动

**症状：** `service cron status` 显示未运行

**解决：**
```bash
sudo service cron start
```

### 问题 2：环境变量未生效

**症状：** `echo $HF_ENDPOINT` 显示空

**解决：**
1. 关闭所有 terminal
2. 重新打开 terminal
3. 或者手动执行：`source ~/.bashrc`

### 问题 3：模型加载失败

**症状：** 日志显示 "Network is unreachable"

**解决：**
1. 检查代理是否正常：`echo $HTTP_PROXY`
2. 检查 HF_ENDPOINT：`echo $HF_ENDPOINT`
3. 重启 terminal 让配置生效

### 问题 4：Crontab 丢失

**症状：** `crontab -l` 看不到配置

**解决：**
```bash
cd ~/.openclaw/agents/personal-assistant/workspace/记忆库/memory-system
cat crontab.txt | crontab -
```

## 📝 可选配置

### OpenClaw Gateway 自启

如果需要 OpenClaw 网关也开机自启，取消 `~/.bashrc` 中的注释：

```bash
# OpenClaw Gateway 自启动（如果未运行）
if ! pgrep -f "openclaw gateway" > /dev/null 2>&1; then
    echo "Starting OpenClaw Gateway..."
    nohup openclaw gateway start > /tmp/openclaw-gateway.log 2>&1 &
fi
```
